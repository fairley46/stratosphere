# Stratosphere Brand Guide

Date: March 2, 2026

## Brand Positioning
Stratosphere is the migration architect for enterprise legacy applications.
It gives teams a clear, auditable path from VM sprawl to Kubernetes-ready architecture without risky manual rewrites.

## One-Line Message
"Interrogate the VM. Explain the system. Generate the migration plan."

## Elevator Pitch
Stratosphere inspects how a production VM behaves in real life, maps the application composition, and generates human-reviewable Kubernetes migration artifacts with governance gates. Teams move faster, with less migration debt and lower execution risk.

## Target Audiences
1. Enterprise application owners who need modernization without downtime surprises.
2. Cloud migration consultants and platform teams delivering repeatable transformations.
3. Security and operations reviewers who require auditability and human controls.

## Value Pillars
1. Clarity:
   - Current-state and future-state maps in plain language.
2. Control:
   - Human review, multi-approver workflow, preflight checks, rollback plans.
3. Speed:
   - Automated discovery, decomposition, and artifact generation.
4. Confidence:
   - Readiness scoring, unknowns tracking, business impact, and ROI output.

## Differentiators
1. Deep runtime interrogation beyond OS-level migration.
2. Decomposition recommendations with confidence and rationale.
3. Artifact bundle for Docker, Helm, Terraform, and governance reports.
4. Dual access model: standalone CLI + MCP server for agent workflows.

## Messaging Style
1. Prefer "plain-language, auditable, controlled migration."
2. Avoid "black box automation" phrasing.
3. Emphasize "human-in-the-loop" and "blue/green safety."
4. Use concrete outcomes: reduced migration debt, reduced VM sustainment load, faster planning cycles.

## Brand Promise
Every Stratosphere run must leave the user with:
1. A clear understanding of how the app works today.
2. A clear view of the recommended future architecture.
3. A reviewable package ready for enterprise sign-off.
