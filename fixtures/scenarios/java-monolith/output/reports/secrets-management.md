## Provider Patterns

### Option 1: Kubernetes Secret (simple)
```bash
kubectl create secret generic <component>-secret \
  --from-literal=MY_SECRET_KEY=<value> \
  -n <namespace>
```
The generated Helm chart already mounts this secret via `envFrom.secretRef`.

### Option 2: External Secrets Operator
```yaml
apiVersion: external-secrets.io/v1beta1
kind: ExternalSecret
metadata:
  name: <component>-secret
spec:
  refreshInterval: 1h
  secretStoreRef:
    name: my-secret-store
    kind: SecretStore
  target:
    name: <component>-secret
  data:
    - secretKey: MY_SECRET_KEY
      remoteRef:
        key: my-app/MY_SECRET_KEY
```

### Option 3: Vault Agent Sidecar
Add the following annotation to the Pod spec in `helm/templates/workloads.yaml`:
```yaml
annotations:
  vault.hashicorp.com/agent-inject: "true"
  vault.hashicorp.com/role: "<app-role>"
  vault.hashicorp.com/agent-inject-secret-app.env: "secret/data/<app>"
```
